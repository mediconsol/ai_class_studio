┌─────────────────────────────────────┐
│  AI의 역할 (엄격히 제한)            │
├─────────────────────────────────────┤
│  ✅ 할 수 있는 것                   │
│  - 관찰된 사실 정리                 │

│  - 보고서 형식 맞추기               │
├─────────────────────────────────────┤
│  ❌ 해서는 안 되는 것               │
│  - 원인 추정                        │
│  - 책임 판단                        │
│  - "~때문에", "~미흡" 등 표현       │
└─────────────────────────────────────┘

강사 멘트
오늘 차시에서 가장 중요한 슬라이드입니다.
AI의 역할은 엄격히 제한됩니다.
할 수 있는 것:
관찰된 사실을 정리하는 것
시간순으로 구조화하는 것
보고서 형식에 맞추는 것
해서는 안 되는 것:
원인을 추정하는 것
책임을 판단하는 것
"~때문에", "~미흡", "~실수" 같은 표현
AI는 **"무슨 일이 있었는가"**만 정리합니다. **"왜 일어났는가"**는 AI의 영역이 아닙니다.
이 원칙을 반드시 기억해주세요.

🔷 개념 이해 (8분)

[슬라이드 5. 사고보고서에서 피해야 할 표현]
화면 문구
❌ 피해야 할 표현

- "~때문에" (원인 추정)
- "~미흡" (책임 암시)
- "~실수" (과실 인정)
- "확인하지 못했다" (부주의 암시)
- "부족했다" (책임 암시)
- "~했어야 했다" (의무 위반 암시)

강사 멘트
사고보고서에서 절대 피해야 할 표현입니다.
"~때문에" - 원인을 추정하는 표현입니다. 원인 조사는 별도로 이루어져야 합니다.
"~미흡", "~실수", "~부족" - 책임을 암시합니다. 사고보고서는 책임을 묻는 문서가 아닙니다.
"확인하지 못했다", "~했어야 했다" - 과실이나 의무 위반을 암시합니다.
이런 표현이 들어가면 나중에 법적으로 불리하게 해석될 수 있습니다.
AI에게도 이런 표현을 쓰지 말라고 명확히 지시해야 합니다.

[슬라이드 6. 사고보고서에서 사용해야 할 표현]
화면 문구
✅ 사용해야 할 표현

- "~시경 관찰됨" (시간 + 사실)
- "~로 확인됨" (객관적 확인)
- "~상태였음" (상태 기술)
- "~을 시행함" (조치 기술)
- "~에게 보고함" (보고 사실)

강사 멘트
대신 이런 표현을 사용해야 합니다.
"~시경 관찰됨" - 시간과 사실만 기술합니다. "~로 확인됨" - 객관적으로 확인된 내용만 씁니다. "~상태였음" - 당시 상태를 기술합니다. "~을 시행함" - 취한 조치를 기술합니다. "~에게 보고함" - 보고한 사실을 기록합니다.
공통점이 보이시죠? **"누가, 언제, 무엇을"**만 있고, **"왜"**는 없습니다.
이게 사실 중심 기술입니다.

[슬라이드 7. 환자안전 보고서의 기본 구조]
화면 문구
■ 사고보고서 기본 구조

1. 사고 개요

2. 사고 발생 상황

강사 멘트
대부분의 환자안전 보고서는 이 구조를 따릅니다.
사고 개요: 무슨 사고인지, 누구에게, 언제 발생 상황: 어디서, 어떤 상황에서, 누가 발견했는지 환자 상태: 사고 직후 환자가 어떤 상태였는지 즉각 조치: 어떤 조치를 취했는지
오늘 실습에서도 이 구조를 그대로 사용합니다.
AI에게 이 구조를 주면, AI가 이 틀에 맞춰서 정리해줍니다.
