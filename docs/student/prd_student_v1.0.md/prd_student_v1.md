# 🧑‍🎓 수강생(Student) 화면 와이어프레임

---

## 1️⃣ 수강생 메인 대시보드

```
┌──────────────────────────────────────────┐
│ 상단바                                    │
│ [로고]  과정명 | 마이페이지 | 로그아웃   │
└──────────────────────────────────────────┘

┌──────────────────────────────────────────┐
│ 📘 나의 차시별 실습 현황                  │
│------------------------------------------│
│ 6차시 간호기록보조   [실습하기] [완료]   │
│ 7차시 인수인계       [실습하기] [미제출] │
│ 8차시 환자상태요약   [실습하기] [제출완료]│
│ ...                                      │
└──────────────────────────────────────────┘
```

**기능**

* 차시별 상태 표시

  * `미실습 / 저장됨 / 제출완료 / 평가완료`
* `실습하기` → 실습 플레이그라운드 이동

---

## 2️⃣ 실습 플레이그라운드 (핵심 화면)

> ✅ 강의용 솔루션과 **UI 100% 동일**

```
┌──────────────────────────────────────────┐
│ [차시명] 7차시_인수인계                  │
│ 설명: 교대 간호 인수인계 요약 실습       │
└──────────────────────────────────────────┘

┌───────────────┬──────────────────────────┐
│ ⬅ 프롬프트    │ ➡ AI 출력 결과            │
│ 입력 영역     │                          │
│---------------│--------------------------│
│ (textarea)    │ (markdown render)        │
│               │                          │
└───────────────┴──────────────────────────┘

┌──────────────────────────────────────────┐
│ [AI 실행]  [저장]  [제출]                │
└──────────────────────────────────────────┘
```

**버튼 동작**

* `AI 실행` : 실시간 실행
* `저장` : 마이페이지에 임시 저장
* `제출` : 해당 차시 과제 제출 (⚠ 1개만 가능)

---

## 3️⃣ 마이페이지 – 저장/제출 관리

```
┌──────────────────────────────────────────┐
│ 마이페이지                                │
└──────────────────────────────────────────┘

┌──────────────────────────────────────────┐
│ 📂 저장된 실습 목록                      │
│------------------------------------------│
│ [7차시] 인수인계 실습  2025-03-01 [선택] │
│ [7차시] 인수인계 실습  2025-03-02 [선택] │
└──────────────────────────────────────────┘

┌──────────────────────────────────────────┐
│ [선택한 실습 미리보기]                   │
│------------------------------------------│
│ 프롬프트 (읽기 전용)                     │
│ 결과물                                  │
│                                          │
│ [이 실습 제출하기]                       │
└──────────────────────────────────────────┘
```

📌 **제출 이후**

* 수정 불가
* 상태 → `제출완료`

---

## 4️⃣ 마이페이지 – 평가 결과 확인

```
┌──────────────────────────────────────────┐
│ 📊 평가 결과                             │
│------------------------------------------│
│ 7차시 인수인계                           │
│ 점수: 85점                               │
│ 평가 코멘트:                             │
│ - 핵심 정보 요약이 명확함               │
│ - 주의사항 강조가 좋음                  │
└──────────────────────────────────────────┘
```

---

# 🧑‍⚖️ 평가자(Reviewer) 화면 와이어프레임

---

## 5️⃣ 평가자 로그인 후 메인

```
┌──────────────────────────────────────────┐
│ 평가자 전용 화면                         │
│ [제출물 관리] [로그아웃]                 │
└──────────────────────────────────────────┘

┌──────────────────────────────────────────┐
│ 📥 제출물 목록                           │
│------------------------------------------│
│ 수강생 | 차시 | 제출일 | 상태 | 보기     │
│------------------------------------------│
│ 홍길동 | 7차시 | 3/2 | 미평가 | [열기]   │
│ 김OO  | 9차시 | 3/3 | 평가완료 | [열기] │
└──────────────────────────────────────────┘
```

**필터**

* 차시별
* 평가 상태별

---

## 6️⃣ 제출물 상세 + 평가 화면 (핵심)

```
┌──────────────────────────────────────────┐
│ 제출물 상세                              │
│ 수강생: 홍길동 | 차시: 7차시 인수인계   │
└──────────────────────────────────────────┘

┌───────────────┬──────────────────────────┐
│ ⬅ 프롬프트    │ ➡ 결과물                 │
│ (읽기 전용)   │ (읽기 전용)              │
└───────────────┴──────────────────────────┘

┌──────────────────────────────────────────┐
│ 📝 평가 입력                             │
│------------------------------------------│
│ 점수: [  85 ] (0~100)                    │
│ 평가 코멘트:                             │
│ [ textarea ]                             │
│                                          │
│ [평가 완료]                              │
└──────────────────────────────────────────┘
```

📌 **중요 원칙**

* 평가자는 **AI 실행 불가**
* 프롬프트·결과물 **읽기 전용**

# 🔐 권한 흐름 요약

```
학생
- 실습
- 저장
- 제출
- 평가 결과 열람

평가자
- 제출물 열람
- 점수/코멘트 입력
```

---

## 🎯 이 와이어프레임의 강점

1. **강의용 UI와 완전 동일 → 학습 혼란 없음**
2. 차시 기반 → 직무 연계성 명확
3. 제출·평가 구조 → 심사 대응 최적
4. SaaS 확장 가능 (교육 플랫폼 / 병원 내부 교육)
