# 📘 PRD – 의료기관 AI 실무 교육 강의용 사이트 (Option B · MVP)

---

## 1. 제품 개요 (Product Overview)

### 1.1 제품명

* **의료기관 AI 실무 교육 강의용 사이트 (가칭: MediConsol AI Education Studio)**

### 1.2 제품 목적

본 제품은 의료기관 종사자를 대상으로 한 **AI 실무 교육 과정**을 효과적으로 운영하기 위한 **강의·촬영·실습 통합형 웹 플랫폼**이다.

단순한 LMS가 아닌,

* 강사가 **촬영하며 시연할 수 있고**
* 수강생이 **실무 흐름을 이해할 수 있으며**
* AI 활용 결과물이 **업무 산출물로 연결되는 구조**를 목표로 한다.

### 1.3 MVP 범위 정의

MVP는 **“차시별 강의 화면 1개”**를 중심으로 다음을 포함한다.

* 슬라이드 기반 강의 화면
* AI 시연 패널
* 기본 자료 열람 기능

---

## 2. 타깃 사용자 (Target Users)

### 2.1 1차 사용자 – 강사

* 의료기관 AI 교육 강사
* 요양병원·의원·병원 실무 전문가

**주요 니즈**

* 촬영에 최적화된 화면
* 실시간 AI 시연 가능
* 강의 흐름이 끊기지 않는 UI

### 2.2 2차 사용자 – 수강생 (MVP에서는 간접 고려)

* 간호사, 행정직, 의료지원 인력
* AI 실무 활용에 익숙하지 않은 사용자

**주요 니즈**

* 실제 업무와 연결된 예시
* 프롬프트 구조 이해
* 결과물 활용 가능성

---

## 3. 핵심 사용자 시나리오 (User Flow)

### 3.1 강사용 핵심 시나리오

1. 강의 사이트 접속
2. 특정 차시 선택 (예: 6차시)
3. 슬라이드 탭에서 강의 진행
4. AI 시연 탭으로 전환
5. 프롬프트 입력 → AI 실행
6. 결과를 화면에 표시하며 설명

---

## 4. 기능 범위 정의 (Scope)

### 4.1 MVP 포함 기능 (In Scope)

#### 4.1.1 차시별 강의 화면

* 차시 제목 표시
* 상단 고정 헤더

#### 4.1.2 탭 기반 콘텐츠 구조

* 슬라이드 탭
* AI 시연 탭
* 자료 탭

#### 4.1.3 슬라이드 뷰어

* 이미지 기반 슬라이드 표시
* 이전/다음 버튼
* 현재 슬라이드 번호 표시

#### 4.1.4 AI 시연 패널

* 프롬프트 입력 영역
* AI 실행 버튼
* AI 응답 결과 표시 영역
* 로딩 상태 표시

#### 4.1.5 AI API 연동

* 서버 경유 방식(OpenAI 기준)
* 단일 모델 사용

---

### 4.2 MVP 제외 기능 (Out of Scope)

* 사용자 로그인/인증
* 학습 진도 저장
* 과제 제출 및 평가
* 관리자 CMS
* 다중 차시 관리 UI

---

## 5. 화면별 요구사항 (Screen Requirements)

### 5.1 차시별 강의 메인 화면

**화면 목적**

* 하나의 차시에서 강의·시연을 모두 수행

**구성 요소**

* 헤더: 차시 제목
* 탭 메뉴: 슬라이드 / AI 시연 / 자료
* 콘텐츠 영역: 선택된 탭 콘텐츠 표시

---

### 5.2 슬라이드 탭

**기능 요구사항**

* 슬라이드 이미지 배열 로딩
* 이전/다음 이동
* 전체 화면 비율 유지

**비기능 요구사항**

* 촬영 시 글자 식별 가능
* 불필요한 UI 최소화

---

### 5.3 AI 시연 탭

**기능 요구사항**

* 자유 텍스트 프롬프트 입력
* 실행 버튼 클릭 시 AI 호출
* 응답 결과 텍스트 출력

**UX 요구사항**

* 좌측: 입력 / 우측: 결과
* 실행 중 상태 명확히 표시

---

### 5.4 자료 탭 (MVP 수준)

**기능 요구사항**

* 고정된 예시 텍스트 표시
* 향후 자료 저장소 확장 고려

---

## 6. 기술 요구사항 (Technical Requirements)

### 6.1 프론트엔드

* React (Vite 기반)
* Tailwind CSS
* 컴포넌트 단위 설계

### 6.2 백엔드

* Node.js + Express
* REST API 구조

### 6.3 AI 연동

* OpenAI API (Chat Completion)
* 서버 단에서 API 키 관리

### 6.4 배포 환경

* Frontend: Vercel
* Backend: Railway 또는 유사 환경

---

## 7. 비기능 요구사항 (Non-Functional Requirements)

### 7.1 성능

* AI 응답 대기 중 UI 멈춤 없음

### 7.2 보안

* API Key 클라이언트 노출 금지
* CORS 설정

### 7.3 확장성

* 차시 단위 컴포넌트 재사용 가능
* Presenter Mode 확장 가능 구조

---

## 8. 성공 기준 (Success Metrics)

### 8.1 MVP 성공 기준

* 강의 촬영 중 오류 없이 AI 시연 가능
* 슬라이드와 AI 전환이 자연스러움
* 실제 교육 영상 촬영 완료

### 8.2 다음 단계 전환 조건

* MVP 기반으로 전체 차시 확장
* 프레젠터 모드 개발 착수

---

## 9. 향후 확장 로드맵 (Post-MVP)

본 서비스는 MVP 이후 **강의 → 실습 → 과제 → 관리**까지 포함하는
**의료기관 AI 실무 교육 전용 플랫폼**으로 확장한다.

---

### Phase 2: 강의·촬영 고도화 (Instructor Experience 강화)

#### 9.1 프레젠터 모드 (Presenter Mode)

**목적**
강사가 촬영 중 강의 흐름을 놓치지 않고 슬라이드·멘트·AI 시연을 통합 제어할 수 있도록 한다.

**주요 기능**

* 전체화면 슬라이드 표시
* 다음 슬라이드 미리보기 패널
* 강사 멘트(스크립트) 표시/숨김
* 현재 차시/슬라이드 타이머 표시
* AI 시연/자료 패널 팝업 호출

**비기능 요구사항**

* 촬영 시 불필요한 UI 완전 제거
* 키보드 단축키 지원 (←/→, Space, F 등)

---

#### 9.2 프롬프트 템플릿 라이브러리

**목적**
AI 시연을 즉흥이 아닌 **구조화된 교육 콘텐츠**로 제공한다.

**주요 기능**

* 차시별 프롬프트 템플릿 등록
* 트랙별 분류 (간호 / 행정 / 통합)
* 프롬프트 미리보기 및 즉시 실행
* 즐겨찾기(핀 고정)

**프롬프트 메타 정보**

* 목적 설명
* 사용 시점 (강의/과제/실습)
* 입력 예시
* 결과 활용 가이드

---

#### 9.3 예제데이터 관리

**목적**
AI 시연과 실습에서 **현실적인 의료 데이터 예시**를 반복 활용할 수 있도록 한다.

**주요 기능**

* 차시별 예제데이터 등록
* 데이터 유형 분류 (간호메모, 민원, 행정문서 등)
* 원클릭 삽입 (프롬프트 입력창으로)
* 수정/복사 가능

---

### Phase 3: 학습자 중심 기능 확장 (Education Platform 완성)

#### 9.4 수강생용 화면 분리

**목적**
강의용(강사) 화면과 학습용(수강생) 화면을 명확히 분리하여 혼란을 방지한다.

**구분 원칙**

* 강사용: 실행·시연 중심
* 수강생용: 이해·복습 중심

**수강생 화면 특징**

* AI 시연 결과는 읽기 전용
* 프롬프트는 복사 가능
* 강의 영상/슬라이드 중심 UI

---

#### 9.5 과제 제출 기능 (16~20차시)

**목적**
수강생이 AI를 활용해 **실제 업무 산출물**을 만들어 제출하도록 한다.

**과제 화면 구성**

* 과제 설명
* 입력 영역 (문제 상황 / 데이터)
* AI 보조 실행 영역
* 결과물 편집
* 제출 (텍스트 / PDF)

**평가 요소 (확장 가능)**

* 구조 적합성
* 기록 문장 품질
* 실무 활용 가능성

---

#### 9.6 관리자 패널 (Admin Console)

**목적**
교육 콘텐츠와 사용자 활동을 중앙에서 관리한다.

**주요 기능**

* 차시/슬라이드/프롬프트 관리
* 예제데이터 관리
* 수강생 과제 제출 현황 조회
* 기본 통계 (차시별 참여도)

---

## 10. 서비스 전체 구조 요약

본 서비스는 단계적으로 다음 구조를 완성한다.

1. **강의 스튜디오**

   * 슬라이드 + AI 시연 + 촬영 최적화

2. **실무 실습 환경**

   * 프롬프트 + 예제데이터 + 결과물 생성

3. **교육 플랫폼**

   * 수강생 화면 + 과제 제출 + 관리자 관리

---

## 11. 최종 지향점

본 서비스는 단순한 교육 사이트가 아니라,

> **의료기관 종사자가 AI로 ‘일을 줄이는 방법’을
> 직접 보고, 따라 하고, 결과물을 남기는 플랫폼**

을 최종 목표로 한다.

이 PRD는 전체 서비스 구현을 위한 **상위 설계 문서**로 사용한다.
